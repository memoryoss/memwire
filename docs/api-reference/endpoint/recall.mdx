---
title: 'Recall context'
description: 'Retrieve relevant memory context for a query'
icon: 'brain'
---

`POST /v1/memory/recall`

### Request body

| Field | Type | Required | Description |
|---|---|---|---|
| `user_id` | string | ✅ | User to recall memory for |
| `query` | string | ✅ | The query to find relevant memories for |
| `app_id` | string | | Filter to a specific application |
| `workspace_id` | string | | Filter to a specific workspace or team |
| `agent_id` | string | | Filter to a specific agent |
| `top_k` | integer | | Max number of memory paths (default: `5`) |

### Example

```bash
curl -X POST http://localhost:8000/v1/memory/recall \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "alice",
    "app_id": "app_a",
    "workspace_id": "team_1",
    "query": "How should I format my answers?",
    "top_k": 5
  }'
```

### Response

```json
{
  "context": "alice prefers dark mode and short answers.",
  "paths": 2,
  "knowledge": []
}
```
